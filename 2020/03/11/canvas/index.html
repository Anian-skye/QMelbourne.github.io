canvas基本使用及动画

Posted on 2020-03-11 | In 前端 , js , canvas

从零开始查资料实现一个小球的动画

[toc]

要求：

  1. canvas 的 宽500 高 500 底色黑色
  2. 创建一个小球 宽20 高20
  3. 点击小球 从 (20, 20) 的地方，向 (480, 480) 由慢到快移动

把要求分解一下：

  1. 建画布（因为踩坑了， 所以单独写一下）
  2. 画球
  3. 事件绑定
  4. 动起来

建画布

踩得坑是：canvas不能使用css来直接设置宽高，这样会让图片变形

比如如果画了个圆，就变成了椭圆；正方形就变成了长方形

个人理解的原因

canvas就像是一张图片，他是基于像素的，他有一个默认大小 300*150

如果使用css来操作这张图片的话，就相当于按比例去放大这张图片

比如我尝试设置500,500 就变成了宽放大（500/300）高放大（500/150），就会得到一个拉长的圆形

解决方法

画球

这一步就没什么好说的了，canvas的操作很像是在画板里面画画

先获取到画板，画笔，然后设置画笔的颜色，起始点和结束点，最后选择是否填充或者描边

事件绑定

canvas的事件绑定

canvas不像svg那样，图形都是dom元素，可以直接进行绑定，canvas需要对点击的位置做判断，看是否处于元素位置

一般来说就是边缘检测的问题（这篇文章总结的很好的样子，有时间学习一下 https://refined-x.com/2019/04/27/canvas-click/）

找到一个简单一点的方法

canvas有一个API叫做 isPointInPath

含义就是判断点是否处于路径中

但是他有一个问题，就是path始终都指的是最后一次绘制的path，如果canvas中有多个闭合的路径，就不能这么使用了

当然也可以进行一下修改

把每个绘制过程放入函数，每次出现点击操作的时候进行重绘，如果发现当前重绘的元素处于点击位置，则触发事件（缺点：需要大量重绘）

动画

MDN上给出的动画步骤

  1. 清空 canvas
    除非接下来要画的内容会完全充满 canvas （例如背景图），否则你需要清空所有。最简单的做法就是用 clearRect 方法。
  2. 保存 canvas 状态
    如果你要改变一些会改变 canvas 状态的设置（样式，变形之类的），又要在每画一帧之时都是原始状态的话，你需要先保存一下。
  3. 绘制动画图形（animated shapes）
    这一步才是重绘动画帧。
  4. 恢复 canvas 状态
    如果已经保存了 canvas 的状态，可以先恢复它，然后重绘下一帧。

个人理解：

canvas的动画过程就是绘制+清空+重画+保存的循环过程

一些变换的操作

平移 translate

旋转 rotate

状态变换

用来状态变换的两个API

restore 和 save

当你执行“变换”操作时，整个上下文的坐标系都将会改变。“变换”之后，我们需要将坐标系恢复到原有正常的状态，这时候就需要使用save()和restore()了。

Canvas Context维持着绘制状态的堆栈。区分绘制状态：

  • 当前矩阵变换例如：平移translate()，缩放scale()，以及旋转rotate()等
  • 剪切区域clip()
  • 以下属性值： strokeStyle，fillStyle，globalAlpha，lineWidth，lineCap，lineJoin，miterLimit，shadowOffsetX，shadowOffsetY，shadowBlur，shadowColor，globalCompositeOperation，font，textAlign，textBaseline。
  • 路径和位图不是绘图状态的一部分，使用save()和restore()不会生效。路径是持久的，只能使用beginPath（）方法重置，位图是画布的属性，而非上下文。
  • context.save()将当前状态压入堆栈。context.restore()弹出堆栈上的顶级状态，将上下文恢复到该状态。

动画

有了状态的保存，变换，恢复，接下来需要的就是动态的循环这个过程就可以得到想要的动画效果了

比如使用setTimeout，但是这里又有一个坑！
setTimeout的运行时间是用户指定的，如果目前大多数设备的屏幕刷新率为60次/秒，其中每个帧的预算时间仅比16毫秒多一点（1秒/ 60 = 16.6毫秒）。但实际上，浏览器有整理工作要做，因此所有工作都需要在10毫秒内完成，所以如果指定的时间无法完成这个操作就会出现掉帧的现象。

而requestAnimationFrame的基本思想就是与浏览器的刷新频率保持同步

他接受一个回调函数作为参数，这个参数会在浏览器重绘之前调用

递归的调用，每绘制一帧再去绘制下一帧
